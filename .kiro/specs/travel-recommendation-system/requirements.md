# 需求文档

## 文档说明

- 文档范围：WayTrip 个性化旅游推荐系统
- 对齐基线：当前仓库实现
- 更新时间：2026-03-26
- 备注：本版重点补充推荐系统、Redis 使用现状与新增浏览行为数据链路

## 产品简介

WayTrip 是一个面向旅游场景的个性化推荐系统，包含：

- 小程序端：Uni-app + Vue 3 + Pinia
- Web 用户端：Vue 3 + Element Plus + Pinia
- 管理端：Vue 3 + Element Plus + ECharts
- 后端：Spring Boot 3 + MyBatis-Plus + MySQL + Redis

系统支持景点浏览、攻略阅读、评分评论、收藏、订单、推荐、个人中心，以及管理端的内容维护和运营统计。

## 术语

- 小程序端：面向移动端用户的 Uni-app 应用
- Web 用户端：面向 PC 用户的 Vue Web 应用
- 管理端：面向管理员的后台系统
- 推荐引擎：基于 ItemCF 的景点推荐逻辑
- 冷启动：用户交互不足时返回热门内容并引导设置偏好
- 逻辑删除：通过 `is_deleted=1` 屏蔽业务数据，不做物理删除
- 浏览行为：来自 `user_spot_view` 的景点访问记录，包含来源与停留时长

## 功能需求

### 需求 1：小程序用户认证

用户可以通过微信登录进入系统；新用户先拿到 `openid`，绑定手机号后完成注册；老用户直接返回登录态。

验收标准：

1. 小程序登录调用 `wx.login` 并换取 OpenID。
2. OpenID 未注册时，仅返回 `openid`，不直接创建完整账户。
3. OpenID 已存在时，返回 token 和用户信息。
4. 绑定手机号后，系统创建用户或将 OpenID 合并到既有账户。
5. 登录后个人中心可读取昵称、头像和偏好信息。

### 需求 2：首页与轮播图

首页需要提供轮播图、热门推荐和个性化推荐入口。

验收标准：

1. 首页展示已启用轮播图。
2. 点击轮播图跳转至对应景点详情。
3. 首页展示热门景点列表。
4. 首页支持下拉刷新或手动刷新推荐内容。

### 需求 3：个性化推荐

系统根据评分、收藏、订单和浏览行为生成个性化推荐；当用户交互不足时使用冷启动策略。

验收标准：

1. 用户交互数达到阈值时返回个性化推荐。
2. 交互不足时返回热门推荐，并提示设置偏好。
3. 推荐结果过滤已评分、已收藏、已下单但未取消的景点。
4. 被取消订单的景点允许重新进入推荐结果。
5. 推荐支持刷新。
6. 浏览行为需要考虑来源与停留时长差异。
7. 推荐结果支持热度重排。

### 需求 4：景点浏览与搜索

用户可浏览、筛选、搜索景点，并查看完整详情。

验收标准：

1. 支持按地区、分类筛选。
2. 支持按热度、评分、价格排序。
3. 支持分页加载。
4. 支持关键字模糊搜索。
5. 景点详情展示图片、简介、票价、开放时间、地址、经纬度、评论、收藏状态。
6. 小程序支持调用地图导航。
7. 浏览行为可写入浏览记录，用于推荐与热度计算。

### 需求 5：旅游攻略

用户可浏览和阅读攻略，查看关联景点。

验收标准：

1. 支持攻略分页列表。
2. 支持按分类筛选。
3. 详情页支持富文本渲染。
4. 展示关联景点并支持跳转。
5. 已下架内容不在用户端展示。

### 需求 6：评分与评论

用户可对景点提交评分和评论。

验收标准：

1. 支持 1-5 分评分。
2. 再次评分时更新原记录。
3. 评论写入后可在详情页读取。
4. 评分写入后同步更新景点平均分和评分数。

### 需求 7：收藏管理

用户可收藏和取消收藏景点。

验收标准：

1. 支持收藏添加。
2. 支持取消收藏。
3. 支持收藏列表查询。
4. 收藏状态变更后前端立即可见。

### 需求 8：订单创建与查询

用户可创建门票订单并查看列表与详情。

验收标准：

1. 创建订单时填写游玩日期、数量、联系人信息。
2. 新建订单默认状态为待支付。
3. 列表支持按状态筛选与分页。
4. 详情展示景点、数量、金额、状态与联系人信息。

### 需求 9：模拟支付

订单支持模拟支付流程。

验收标准：

1. 待支付订单可发起支付。
2. 支付成功后状态更新为已支付。
3. 多次触发支付时不产生重复结果。

### 需求 10：用户取消订单

用户可取消待支付或已支付订单。

验收标准：

1. 允许取消待支付和已支付订单。
2. 已完成订单不可取消。
3. 重复取消已取消订单时按幂等处理。

### 需求 11：管理端订单流转

管理员可查看订单并对订单执行完成、退款、取消、恢复操作。

验收标准：

1. 已支付订单可标记完成。
2. 已支付订单可执行退款。
3. 待支付订单可由管理员取消。
4. 已完成订单可恢复为已支付。

### 需求 12：个人中心

用户可在个人中心查看与管理账户信息。

验收标准：

1. 展示昵称、头像、偏好标签。
2. 提供订单入口。
3. 提供收藏入口。
4. 提供客服入口。
5. 支持头像上传、修改密码、注销账户。

### 需求 13：管理员认证

管理员通过后台登录后访问受保护页面。

验收标准：

1. 后台登录成功后返回管理员 token。
2. 未认证访问后台时重定向或返回认证错误。
3. 管理员密码使用 BCrypt 存储。
4. 被禁用管理员不可登录。

### 需求 14：景点管理

管理员可维护景点基础数据。

验收标准：

1. 支持景点创建与编辑。
2. 支持地区、分类、经纬度、封面图、详情图录入。
3. 支持发布和下架。
4. 支持逻辑删除。
5. 支持后台筛选、搜索和分页。

### 需求 15：攻略管理

管理员可维护攻略内容。

验收标准：

1. 支持攻略创建与编辑。
2. 支持封面图和富文本内容。
3. 支持发布和下架。
4. 支持关联景点。

### 需求 16：管理端订单查询

管理员可检索与查看订单。

验收标准：

1. 支持按状态、时间范围、订单号筛选。
2. 支持分页。
3. 支持订单详情查看。

### 需求 17：用户管理

管理员可查看用户信息并重置密码。

验收标准：

1. 列表展示昵称、头像、手机号脱敏、订单数、评分数、收藏数。
2. 用户详情展示偏好和近期订单。
3. 支持重置用户密码。
4. 后台不展示 OpenID。

### 需求 18：非功能要求

系统需满足基础安全、可靠性和可维护性要求。

验收标准：

1. 用户端和管理端均使用 JWT。
2. 业务读取默认过滤逻辑删除数据。
3. 用户端默认仅展示已发布内容。
4. 订单相关接口具备幂等保护。
5. 推荐相似度矩阵支持 Redis 存储。
6. 推荐配置、运行状态与结果缓存需可观测、可调试。

### 需求 19：仪表板

管理员可查看运营概览与趋势。

验收标准：

1. 展示总用户、总景点、总订单、总收入。
2. 展示今日新增用户、今日订单、今日收入。
3. 展示订单趋势图。
4. 展示热门景点排行。

### 需求 20：Web 用户端认证

Web 用户通过手机号和密码注册登录。

验收标准：

1. 支持手机号注册。
2. 支持手机号密码登录。
3. 手机号重复注册时返回错误。
4. 小程序绑定手机号时可合并 Web 账户。

### 需求 21：管理员管理

超级管理员可管理后台管理员账户。

验收标准：

1. 支持管理员列表查询。
2. 支持新增管理员。
3. 支持更新管理员资料。
4. 支持重置管理员密码。
5. 不允许删除自己。

### 需求 22：账户安全

用户可管理账户安全信息。

验收标准：

1. 修改密码时校验旧密码或允许首次设置密码。
2. 头像上传限制图片类型与大小。
3. 注销账户执行逻辑删除。

### 需求 23：订单超时自动取消

系统自动取消超时未支付订单。

验收标准：

1. 待支付超过 5 分钟自动取消。
2. 定时任务每分钟运行。
3. 自动写入取消时间。

### 需求 24：景点地区管理

管理员维护二级地区树。

验收标准：

1. 支持按父级查询。
2. 支持新增、编辑、删除。
3. 按排序字段展示。

### 需求 25：景点分类管理

管理员维护景点分类树与分类图标。

验收标准：

1. 支持按父级查询。
2. 支持新增、编辑、删除。
3. 支持图标 URL。
4. 按排序字段展示。

### 需求 26：推荐配置与调试

管理员可在后台查看推荐状态、调整参数并验证结果。

验收标准：

1. 支持读取和保存推荐配置。
2. 配置按 `algorithm / heat / cache` 三段组织。
3. 支持查看推荐引擎运行状态。
4. 支持手动重建相似度矩阵。
5. 支持预览指定用户的推荐结果。
6. 支持预览指定景点的相似邻居。
